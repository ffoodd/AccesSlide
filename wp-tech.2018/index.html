Bref, c’est bien. Testez.

Définitions

🤖 Test automatisé
Test dirigé par un automate.
Note : les bookmarklets et extensions navigateurs relèvent du test manuel.
🎯 Front-end
Un document HTML, disponible via un chemin, éventuellement saupoudré de CSS et JS.
🗺 Plan de test
Stratégie adoptée pour déterminer quels sont les tests à effectuer, la qualité ciblée, les intervenants, les outils, le cycle de vie, les livrables, le référentiel… en fonction d’un objectif précis.
📈 Budget
Le budget est primordial afin de préciser un périmètre à cibler — et ne pas devenir un nazi de la qualité.

Les bons plans

Le bon plan ?
C’est un plan, il a un test, il voit une erreur, il crash…

Source inconnue

Deux axes pour un bon plan :

  1. Les résultats doivent être intelligibles ;
  2. un référent doit être identifiable.

Sans ça, vous pissez dans un violon 🎻.

Le cambouis ⚙

Par l’axe du quoi

Précisions :

  • Où : dans le navigateur (décapité ?), dans l’IDE, dans les tâches automatisées, dans une VM, etc.
  • Quand : dès la modification du code, au commit, au merge, au déploiement, en continu sur la prod…
  • Avec quoi : un outil par domaine, un SaaS tout-en-un…
  • Comment gérer les résultats : flux JSON, rapports PDF ou markdown, création de tickets à la volée, alerte Slack, etc.

Entrons dans le vif du sujet !

Qualité de code 🤖

Validator

Fourni par le w3c  :

  • Validation HTML ;
  • Validation CSS ;
  • Validation RSS ;
  • Validation des liens ;
  • Validation de l’i18n.

Il existe aussi des interfaces Node : pour HTML et pour CSS.

Statistiques

Des indicateurs secondaires pour évaluer la complexité du code HTML et CSS :

  • DOM Stats par John Otander ;
  • StyleStats par Koji Ishimoto.

Accessibilité 👽

Disponible :

  • en cli ;
  • en extension navigateur ;
  • branché à votre intégration continue ;
  • dans LightHouse, Protractor…

Tenon et Access Monitor

Service payant, via api. Access Monitor permet d’intégrer Tenon à WordPress :

  • un cpt pour les rapports ;
  • planification ;
  • maintenu par Joe Dolson.

Sécurité 🔒

ZAP  ⚡

Logiciel développé par l’owasp. Il dispose d’une api qu’on peut consommer grâce à un client Node.

🚨 Attention, ça pique !

Observatory

Mozilla propose un scanner qui vérifier les certificats SSL et les entêtes HTTP.

Il existe une interface de commande Node.

Performances 🚀

Phantomas

C’est un collecteur de métriques 📏 en cli, qui utilise PhantomJS 👻.

Louis est une tâche Gulp basée sur Phantomas qui compare un métrique à un budget défini.

Yellow Lab Tools s’appuie sur Phantomas.

Alternative

psi est une version cli du Page Speed Insight, il vous donne même les notes !
